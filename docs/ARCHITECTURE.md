## 项目概述

### 1.1 项目定位

Mem0 是一个智能记忆层，为 AI 助手和代理提供持久化记忆能力。它使 AI 能够：

- 记住用户偏好
- 适应个人需求
- 随时间持续学习

### 1.2 核心特性

| 特性 | 描述 |
|------|------|
| **多模式支持** | Platform（云服务）、Open-Source（自托管）、Server（增强服务器） |
| **三层记忆架构** | L0（快速文件）、L1（结构化）、L2（向量存储） |
| **多代理支持** | 按代理隔离的记忆集合 |
| **语义搜索** | 基于向量相似度的智能检索 |
| **自动回忆** | 自动注入相关记忆到对话上下文 |
| **自动捕获** | 自动存储关键事实 |

### 1.3 技术栈

| 层级 | 技术 |
|------|------|
| **后端 API** | FastAPI (Python 3.9+) |
| **前端 Plugin** | TypeScript (Node.js) |
| **向量存储** | PostgreSQL + pgvector |
| **图存储** | Neo4j (可选) |
| **缓存** | Redis |
| **LLM** | OpenAI 兼容 API |
| **嵌入模型** | text-embedding-3-small / bge-m3 |
| **容器化** | Docker + Docker Compose |

## 核心组件

### 3.1 Mem0 Server (FastAPI)

**职责**：提供 REST API 接口

| 组件 | 功能 |
|------|------|
| **FastAPI 应用** | HTTP 服务器、路由、中间件 |
| **API 认证** | X-API-Key header 验证 |
| **速率限制** | Redis 滑动窗口 |
| **多代理实例池** | 按 agent_id 隔离的 Memory 实例 |
| **CORS 中间件** | 跨域资源共享 |

**端点**：
- `GET /health` - 健康检查
- `POST /admin/keys` - 创建 API Key
- `GET /admin/keys` - 列出 API Key
- `DELETE /admin/keys` - 撤销 API Key
- `POST /configure` - 配置 Mem0
- `POST /memories` - 创建记忆
- `GET /memories` - 获取所有记忆
- `GET /memories/{id}` - 获取单个记忆
- `POST /search` - 搜索记忆
- `PUT /memories/{id}` - 更新记忆
- `DELETE /memories/{id}` - 删除记忆
- `GET /memories/{id}/history` - 获取记忆历史

### 3.2 OpenClaw Plugin (TypeScript)

**职责**：为 OpenClaw 平台提供记忆能力

| 组件 | 功能 |
|------|------|
| **ServerClient** | HTTP 客户端，与 Server API 通信 |
| **L0Manager** | L0 层 memory.md 文件管理 |
| **L1Manager** | L1 层日期/分类文件管理 |
| **PlatformProvider** | Mem0 云服务支持 |
| **OSSProvider** | 本地自托管支持 |
| **ServerProvider** | 增强服务器支持 |

**工具**：
- `memory_search` - 搜索记忆
- `memory_list` - 列出记忆
- `memory_store` - 存储记忆
- `memory_get` - 获取单个记忆
- `memory_forget` - 删除记忆
- `memory_l0_update` - 更新 L0 记忆
- `memory_l1_write` - 写入 L1 记忆

### 3.3 Mem0 Core Library

**职责**：核心记忆管理逻辑

| 类 | 功能 |
|------|------|
| **Memory** | 同步记忆操作 |
| **AsyncMemory** | 异步记忆操作 |
| **MemoryClient** | 云平台客户端 |
| **AsyncMemoryClient** | 异步云平台客户端 |
| **FactExtraction** | LLM 事实提取 |
| **Embedding** | 向量嵌入生成 |
| **VectorSearch** | 向量相似度搜索 |
| **Reranker** | 搜索结果重排序 |

### 3.4 存储组件

| 组件 | 类型 | 用途 |
|------|------|------|
| **PostgreSQL + pgvector** | 向量数据库 | 存储 L2 层记忆向量 |
| **Redis** | 内存缓存 | 速率限制、搜索缓存 |
| **Neo4j** | 图数据库 | 实体关系存储（可选）|
| **SQLite** | 关系数据库 | 记忆变更历史 |
| **文件系统** | 本地文件 | L0 memory.md、L1 分类文件 |

### 4.1 记忆创建流程

```
客户端请求 (POST /memories)
         │
         ▼
    [API 认证]
    验证 X-API-Key
         │
         ▼
    [速率限制检查]
    Redis 滑动窗口
         │
         ▼
    [获取代理实例]
    按 agent_id 隔离
         │
         ▼
    [LLM 事实提取]
    从消息中提取结构化事实
         │
         ▼
    [生成向量嵌入]
    使用嵌入模型
         │
         ▼
    [存储到向量数据库]
    pgvector 集合
         │
         ▼
    [记录历史]
    SQLite history.db
         │
         ▼
    [返回结果]
    { results: [...], relations: [...] }
```

```
客户端请求 (POST /search)
         │
         ▼
    [API 认证]

         │
         ▼
    [速率限制检查]
    Redis 滑动窗口
         │
         ▼
    [获取代理实例]

         │
```

## 三层记忆架构

### 5.1 架构概览

```
┌─────────────────────────────────────────────────────────────────┐
│                    三层记忆架构                              │
│  ────────────────────────────────────────────────────      │
│                                                           │
│  ┌───────────────────────────────────────────────────────┐  │
│  │ L0: 持久记忆层 (Persistent Layer)                   │  │
│  │  · 单个 memory.md 文件                               │  │
│  │  · 最快的访问速度                                   │  │
│  │  · 存储最重要的用户事实                             │  │
│  │  · 人类可读的 Markdown 格式                          │  │
│  └───────────────────────────────────────────────────────┘  │
│                         │                                  │
│  ┌───────────────────────────────────────────────────────┐  │
│  │ L1: 结构化层 (Structured Layer)                    │  │
│  │  · 日期文件：2026-03-07.md                        │  │
│  │  · 分类文件：projects.md, contacts.md 等             │  │
│  │  · 提供最近的对话上下文                             │  │
│  │  · 按类别组织的信息                                 │  │
│  └───────────────────────────────────────────────────────┘  │
│                         │                                  │
│  ┌───────────────────────────────────────────────────────┐  │
│  │ L2: 向量层 (Vector Layer)                          │  │
│  │  · PostgreSQL + pgvector                             │  │
│  │  · 语义相似度搜索                                   │  │
│  │  · 存储所有提取的事实                               │  │
│  │  · 支持图记忆（Neo4j）                             │  │
│  └───────────────────────────────────────────────────────┘  │
│                                                           │
└─────────────────────────────────────────────────────────────────┘
```

### 5.2 L0 层：持久记忆

| 属性 | 值 |
|------|------|
| **存储** | 单个 Markdown 文件 |
| **路径** | 默认 `./memory.md` |
| **格式** | Markdown 列表 |
| **访问速度** | ~10ms |
| **容量** | 推荐 <1000 条事实 |
| **用例** | 用户偏好、重要事实 |

**示例内容**：
```markdown
# Memory

> This file contains important facts and information about you.
> It is automatically maintained by memory system.

- User prefers dark mode
- User is allergic to peanuts
- User works as software engineer
- User lives in San Francisco
```

### 5.3 L1 层：结构化记忆

| 属性 | 值 |
|------|------|
| **存储** | 日期文件 + 分类文件 |
| **路径** | 默认 `./memory/` |
| **格式** | Markdown |
| **访问速度** | ~20-50ms |
| **日期文件** | 最近 N 天的对话摘要 |
| **分类文件** | projects, contacts, tasks 等 |

**目录结构**：
```
memory/
├── 2026-03-05.md
├── 2026-03-06.md
├── 2026-03-07.md
├── projects.md
├── contacts.md
└── tasks.md
```

### 5.4 L2 层：向量记忆

| 属性 | 值 |
|------|------|
| **存储** | PostgreSQL + pgvector |
| **集合隔离** | 按 agent_id 隔离 |
| **向量维度** | 1024 (bge-m3) 或 1536 (text-embedding-3-small) |
| **索引** | HNSW (分层小世界图) |
| **访问速度** | ~80-100ms |
| **容量** | 数百万条记录 |

**数据结构**：
```sql
CREATE TABLE memories (
    id VARCHAR PRIMARY KEY,
    memory TEXT NOT NULL,
    embedding vector(1024),
    user_id VARCHAR,
    agent_id VARCHAR,
    run_id VARCHAR,
    metadata JSONB,
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

CREATE INDEX ON memories USING hnsw (embedding vector_cosine_ops);
```

### 5.5 层间协调

| 场景 | 使用的层 | 策略 |
|------|-----------|--------|
| **快速事实查询** | L0 | 直接读取 memory.md |
| **最近对话** | L1 | 读取最近日期文件 |
| **语义搜索** | L2 | 向量相似度搜索 |
| **全面上下文** | L0+L1+L2 | 合并所有层结果 |
| **自动回忆** | L0+L1+L2 | 注入 System Prompt |

## 安全模型

### 6.1 认证机制

```
┌─────────────────────────────────────────────────────────────────┐
│                      认证流程                                │
│                                                           │
│  客户端请求 ──→ [检查 X-API-Key header]                    │
│       │                                                     │
│       ├── 无 Header ──→ HTTP 401 Unauthorized              │
│       │                                                     │
│       ├── 有 Header                                        │
│       │      │                                             │
│       │      ├── 管理端点 (/admin/*)                       │
│       │      │      ├── 验证 ADMIN_SECRET_KEY                 │
│       │      │      └── 匹配 → 允许访问                    │
│       │      │                                             │
│       │      └── 普通 API 端点                              │
│       │             ├── 从文件加载 API Keys                    │
│       │             ├── 查找并验证 Key                        │
│       │             ├── 检查 revoked 状态                     │
│       │             └── 有效 → 允许访问                       │
│                                                           │
└─────────────────────────────────────────────────────────────────┘
```

### 6.2 速率限制

| 参数 | 默认值 |
|------|--------|
| **算法** | 滑动窗口 |
| **窗口大小** | 60 秒 |
| **请求限制** | 200 请求 |
| **存储** | Redis Sorted Set |

**Redis 实现**：
```
ZREM key 0 (now - window)  # 移除过期请求
ZCARD key                  # 计数当前请求
ZADD key {uuid} now        # 添加新请求
EXPIRE key window           # 设置过期时间
```

### 6.3 多代理隔离

| 隔离级别 | 机制 |
|----------|------|
| **集合级别** | `memories_{agent_id}` |
| **实例级别** | 每个 agent_id 独立 Memory 实例 |
| **API Key 绑定** | Key → agent_id 关联 |
| **查询范围** | 仅限绑定的 agent_id |

**集合命名**：
- Agent `my-agent` → `memories_my_agent`
- Agent `test-123` → `memories_test_123`
